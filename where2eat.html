Where2Eat

Image-based machine learning restaurant recommendations

Overview

We have a problem.

We don't know where to eat next and we want to develop a unique solution to this problem. Thus we decided on a fully image-based (as opposed to search-based) recommendation web application that uses image recognition and machine learning to classify photos of food and learn what the user likes.

No need for typing or searching. Just click and go.

We no longer have a problem.

This idea came about at CalHacks, one of the largest hackathons in the world.

My team and I were brainstorming ideas for applications we would use and realized that we would love a restaurant recommendation service. As opposed to Yelp, where we manually search up what we're in the mood for to get reccomendations accordingly, Where2Eat would automatically suggest a restaurant from your preferred cuisine and within your ideal price range.

We used Microsoft's computer vision custom AI to classify pictures of food. We labeled images of food with tags and used these labels to develop an ML model for the user using Synaptic, a JavaScript ML library.

The machine would then use images of restaurants' foods and suggest the restaurant with the highest score according to the user's ML model.

Technology

Technologies Used:

  1. NodeJS
  2. MongoDB (MLAB)
  3. Express
  4. React or AngularJS
  5. Synaptic (Machine Learning JavaScript Library)
  6. Microsoft Computer Vision CustomAI (Machine Learning Image Recognition)
  7. GooglePlacesAPI

How?

  1. Use machine learning to predict what restaurants the user would like to try in a new area based off of:
    • Images the user selects - we use preclassified images to lower processing load on Microsoft
    • From these labels, we form a model and import images from random restaurants
    • Use Microsoft customAI image recognition to label the images
    • Choose a restaurant based on label/model score with Synpatic JS ML
    • Improve model with user feedback
    • Store user models in MongoDB through MLAB
  2. Front End User-Flow
    • 3x3 grids of photos for preference building and model initiation
      • Users can keep picking photos
      • This data trains our neural network
      • Users can indicate diet preferences (vegetarian/vegan) (allergies optional)
      • Users can return to this at anytime to update their preferences
    • Machine then gives suggestions
      • Show multiple photos
      • Show Name
      • Show location on map
      • Show Reviews from (GoogleAPI - net yet implemented)
    • Possible Responses:
      • Go there! (validates machine)
      • Maybe another time (validates machine)
      • Nah (invalidates machine)
  3. Restaurants can be saved (optional)
    • User can create custom lists. For example: "dessert/authentic/etc"
  4. Can predict for multiple people (optional)
    • User can add friends and the app will use their data as well for group decisions

Galactic Weather App

UI Design

Overview


This was a project for my UI design class. The assignment was to design an interface for a mobile app that a galactic traveler would use while preparing for a voyage.

The user would have to be able to query the weather at any of the 9 planets in order to prepare their route. They would also have to get alerts about any major galactic weather events so as to plan their voyage accordingly.


I had a lot of fun designing this one! I went for a futuristic space-age feel. On my home screen, I used a circular carousel to imitate the planets revolving around the sun.

When you click on a planet, it takes you to the weather report which includes the forecast for the upcoming week as well as any major weather events. You can then slide through the planets at the top to easily navigate between them. You can also click the button next to a planet's name to add it to a custom screen for easy comparison between planets.

Design Process

Formal Analysis

To begin, I conducted a formal analysis on sci-fi images found online to discern common themes and motifs in typical sci-fi scenes. I noticed that by using dark colors in the background, the image is made to seem vast, which creates a sense of wonder. These dark colors contrast highly with the bright colors of the foreground text, which draws attention to them, making the text seem important. This is further emphasized by the use of short phrases, which gives the text a sense of urgency, as well as the use of uppercase letters, which also make the text seem important. Lastly, the users are pictured interacting with a large display, which further emphasizes the sense of vastness and paints a picture of immersion, as the user is pictured interacting completely with the device.


Prototyping

Based on these insights, I started designing low-fidelity interaction sketches that reflected these principles. After creating a few different interaction sketches, I decided to pursue this one:

Some of the pain points in the first sketch were the fact that there was no way to see multiple planets’ weather forecasts on one screen to compare them, and no way to save certain locations so you could quickly access them. It would be useful for a traveler who frequently visits Pluto, Venus, and Saturn, for example, to have easy access to those planets’ forecasts moreso than the other planets. So, I refined my sketch to create an initial wireframe.

In this wireframe, I added a footer bar that contains buttons for current location, search, and a list of your chosen locations just for easy access and comparison. I expanded on that by creating buttons on each planet’s page to add that planet to your list or remove it if it’s already there. I also created a search page very similar to the one on the iPhone as well as a screen for the list of planets, where you can swipe left to remove it from the list.


Initial Wireframe

User Testing

I began by conducting a heuristic analysis with some users and identified the following violations:

Rating Heuristic Description
2 Recognition vs. Recall Search bar doesn’t have autocomplete or suggestions. Inconvenient because user would have to type out ‘Jupiter’ even though it is the only planet starting with ‘J’. Autocomplete/saved previous searches makes this faster.
3 Visibility of system status No alert/spinner etc. to let the user know what’s happening while waiting to fetch GPS coordinates or working on something else.
1 Aesthetic and minimalist design It may potentially be overwhelming to a user to see all 9 planets listed on one home screen. Instead it may be more aesthetic to have a scroll.

Reiteration

I incorporated the changes suggested during the heuristic evaluation into a second version of my wireframe. In this version, I changed the home screen to scroll through the planets in a circular manner instead of listing them all statically on one screen (aesthetic and minimalist design). I also added an autocomplete function to the search bar to speed up user flow (recognition vs recall). Although not pictured in the wireframe, I added a spinner to show that the app is waiting for GPS signal (visibility of system status). Lastly, although not mentioned in the heuristic evaluation, I realized that there needed to be a way to easily delete planets from the customized list, but not without warning users first, so I created a swipe left to bring up the delete box (user control and freedom).
